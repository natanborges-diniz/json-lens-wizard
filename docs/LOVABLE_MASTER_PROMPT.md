# Lovable Master Prompt

Objetivo: evoluir o projeto json-lens-wizard para uma ferramenta de comparação e venda assistida de lentes no balcão, baseada em catálogo multi-fornecedor confiável.

Diretrizes:
- não tratar catálogo bruto como verdade final
- preservar separação entre documento fonte, estrutura por fornecedor, camada canônica e preço
- comparar primeiro por categoria clínica, categoria comercial, benefícios percebidos e apoio visual
- usar Essilor, Hoya e Zeiss como fornecedores-base
- não forçar equivalência fina quando a confiança for baixa
- destacar sempre o que é explícito e o que é inferido

Prioridades:
1. consolidar base multi-fornecedor
2. criar interface comparativa
3. incluir camada visual forte para campo visual, tratamentos, blue/UV, fotocromia e estética do material
4. apoiar argumento comercial simples no balcão
5. bloquear recomendação final baseada em dados não revisados

Resultado esperado:
- uma interface comparativa clara
- rápida para uso comercial
- visualmente didática
- confiável o suficiente para orientar a venda
